El módulo de **Gestión de usuarios** concentra el alta, edición, activación y baja de todos los usuarios de Betree60. Está disponible únicamente para el rol `admin` y, en modo lectura, para `comercial`.

## Tipos de usuario que se administran aquí

| Tipo | Ruta destino | Quién puede crearlo |
| --- | --- | --- |
| Interno (`admin`, `juridico`, `cobranza`, `comercial`) | `/admin` | `admin` |
| Vendedor (director, manager, advisor, broker) | `/seller` | `admin` o `comercial` |
| Cliente | `/client` | Automático (al cerrar venta) o `admin` |

## Alta de un usuario interno

<Steps>
  <Step title="Abra Gestión de usuarios → Internos → Nuevo">
    Se despliega el formulario de alta.
  </Step>
  <Step title="Capture los datos">
    Nombre completo, correo, teléfono y rol o roles (multi-selección: `admin`, `juridico`, `cobranza`, `comercial`).
  </Step>
  <Step title="Envío del correo de activación">
    Betree60 envía por **Resend** un correo con enlace de activación (válido por 24 horas) para que el usuario defina su contraseña.
  </Step>
  <Step title="Activación de segundo factor">
    En el primer inicio de sesión, si el rol lo requiere, se le solicita configurar 2FA por correo.
  </Step>
</Steps>

## Alta de un vendedor

Un vendedor casi siempre proviene de un **Aspirante** aprobado, pero también puede crearse manualmente:

1. **Vía aspirante aprobado**: al aprobar el expediente, el sistema convierte al aspirante en vendedor y solicita su **posición en la jerarquía** (director / manager / advisor / broker) y su **manager superior**.
2. **Alta manual**: `Nuevo vendedor` → capturar datos, jerarquía y tabulador de comisiones.

<Note>
  Los brokers externos requieren además un **contrato de intermediación** cargado en el expediente antes de poder generar apartados.
</Note>

## Alta de un cliente

Los clientes se crean automáticamente al cerrarse la venta. El alta manual desde Administración solo se usa para casos especiales (migración de cartera histórica, correcciones). Requiere:

- Datos personales y de contacto.
- Semanas asignadas.
- Esquema de pago acordado.
- Documentos base del expediente.

## Editar y desactivar

- **Editar**: modifica datos de contacto, roles y jerarquía. No modifica correo salvo excepción manual desde base de datos.
- **Desactivar**: bloquea el acceso sin borrar información histórica. Betree60 conserva la trazabilidad completa (auditoría) de todo lo que hizo la cuenta.
- **Reactivar**: disponible en cualquier momento por un `admin`.

<Warning>
  Nunca se eliminan usuarios de forma dura. La baja siempre es lógica para preservar la integridad de expedientes, contratos y pagos.
</Warning>

## Auditoría

Cada cambio queda registrado (quién, cuándo, qué campo) y es consultable desde el detalle del usuario. Los cambios de rol y las bajas se notifican por correo al usuario afectado.
